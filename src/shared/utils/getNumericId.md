# Функція getNumericId

## Функція getNumericId — це хеш-функція, яка перетворює текстовий рядок (у нашому випадку UUID правила) у стабільне ціле число.

Ось детальний розбір, чому це важливо і як це працює математично:

1. Навіщо це потрібно?

- API браузера chrome.declarativeNetRequest вимагає, щоб кожне правило мало поле id, яке є числом (integer). Оскільки в нашому інтерфейсі та сторі ми використовуємо UUID ("abc-123-..."), нам потрібно якось перетворити цей текст у число.
- Важливо: Це число має бути стабільним. Якщо ми перезавантажимо розширення, UUID "abc-123" має знову перетворитися на той самий ID, щоб Chrome міг оновити існуюче правило, а не створювати дублікат.

2. Як працює код (алгоритм)

```typescript
    1 function getNumericId(uuid: string): number {
    2   let hash = 0
    3   for (let i = 0; i < uuid.length; i++) {
    4     const char = uuid.charCodeAt(i)
    5     // 1. Алгоритм змішування (аналог Java hashCode)
    6     hash = (hash << 5) - hash + char
    7     // 2. Приведення до 32-бітного цілого числа
    8     hash |= 0
    9   }
   10   // 3. Обмеження діапазону для Chrome
   11   return (Math.abs(hash) % 2147483646) + 1
   12 }
```

## Крок за кроком:

1.  Цикл for: Проходить по кожному символу вашого UUID.
2.  hash << 5 - hash: Це дуже швидкий бінарний спосіб написати hash \* 31. Це стандартний алгоритм (як у Java String.hashCode()), який "змішує" біти символів, щоб мінімізувати колізії (коли різні рядки дають однакове число).
3.  hash |= 0: Оскільки в JavaScript всі числа — це 64-бітні флоати, ця операція примусово перетворює число у 32-бітне ціле (Signed 32-bit Integer). Це гарантує, що ми працюємо в межах пам'яті, які розуміє комп'ютер на низькому рівні.
4.  Math.abs(hash): Хеш може вийти від'ємним, а Chrome приймає тільки додатні ID.
5.  % 2147483646: Це операція остачі від ділення на максимальне значення 32-бітного числа. Вона гарантує, що наш ID ніколи не перевищить ліміт Chrome (2,147,483,647).
6.  (+ 1): Chrome не дозволяє ID рівний 0. Це гарантує, що мінімальний ID буде 1.

## Підсумок:

- Ми використовуємо стабільну хеш-функцію для детермінованого мапінгу UUID-рядків у Integer-ідентифікатори, які вимагає Chrome DNR API.
- Це дозволяє нам коректно синхронізувати стан між нашим React-стором та двигуном фільтрації браузера без втрати ідентифікації правил при оновленнях.
